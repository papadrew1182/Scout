## What shipped in this PR

### Migrations (mirrored to `backend/migrations/` and `database/migrations/`)

All three files are transactional (`BEGIN; ... COMMIT;`) and idempotent (`IF EXISTS` / `IF NOT EXISTS` throughout).

**`053_phase1_drop_dependent_views.sql`** - drops 10 scout.* views.

- **8 shim views** from `022_session2_canonical.sql:183-190`: `scout.families`, `scout.family_members`, `scout.user_accounts`, `scout.sessions`, `scout.role_tiers`, `scout.role_tier_overrides`, `scout.connector_mappings`, `scout.connector_configs`. These `SELECT * FROM public.X`; PR 1.3 drops the underlying public tables, which Postgres blocks while a view still depends on them. The plan forbids CASCADE.
- **2 named views** over rebuild targets: `scout.v_household_today` (022:761, joins `scout.task_occurrences` + `scout.task_templates` + `public.family_members`), `scout.v_rewards_current_week` (022:794, joins `public.family_members` + `scout.allowance_periods` + `scout.allowance_results`).
- Not dropped: `scout.v_calendar_publication` (022:820) and `scout.v_control_plane` (022:839). Both reference only retained scout.* tables; neither has a dependency on Phase 1 drop targets. Ambiguity 1 findings approved by Andrew during PR 1.1 execution.

**`054_phase1_drop_fks_on_retained.sql`** - 65 FK drops across 44 retained scout.* tables.

Structured in two sections so the migration is self-documenting about what pre-flight Part 1 covered versus what it missed:

- **Section A: 38 FK drops on 24 retained tables** enumerated in pre-flight Part 1's matrix. Part 1's summary sentence said "23 of 26 retained tables have FKs"; the matrix itself actually contained drop-worthy FKs on 24 tables (Part 1 classified `scout.nudge_dispatch_items` as self-contained, but its own row for that table listed `family_member_id -> public.family_members` with "drop FK + recreate"). Individual FK count summed across those 24 tables is 38.
- **Section B: 27 FK drops on 20 additional retained scout.* tables** that Part 1 did not enumerate. Surfaced during PR 1.1 execution by full pg_dump scan. Ambiguity 4 of the pre-execution ambiguity surface was expanded by Andrew to cover this full set. See "Open items" below for Phase 2 recreate-ownership gaps.

Section A/B sub-groupings inside 054 annotate each table's Phase 2 status (in PR 2.5 Build list; in PR 2.2 Build list; purely retained). Section B.1 is 6 FKs; B.2 is 3 FKs; B.3 is 18 FKs.

**`055_phase1_drop_scout_rebuilt.sql`** - drops 5 scout.* tables being rebuilt fresh in Phase 2 PR 2.2.

- `scout.routine_steps` - added to the drop set during PR 1.1 (Ambiguity 2 expansion). Its FK to `scout.routine_templates` (ON DELETE CASCADE) blocks the plan's forbid-CASCADE rule; dropping it here lets PR 2.2's `CREATE TABLE scout.routine_steps (...)` re-establish both FKs (`routine_template_id` + `standard_of_done_id`) cleanly.
- `scout.task_occurrences`, `scout.task_templates`, `scout.routine_templates`, `scout.meal_transformations` - per v5.1 §Phase 1 PR 1.1 original scope. `meal_transformations` is a pure drop (not rebuilt) per v5.1 §2 Q7.

Drop order inside 055 is `routine_steps -> task_occurrences -> task_templates -> routine_templates -> meal_transformations`, which releases inter-table FKs naturally without CASCADE after migration 054 has dropped the FKs on retained tables pointing into this set.

### What migration 054 intentionally does NOT cover

- **FKs on retained scout.* tables that point at other retained scout.* tables** (e.g. `scout.nudge_dispatch_items.dispatch_id -> scout.nudge_dispatches`, `scout.push_deliveries.push_device_id -> scout.push_devices`, `scout.affirmation_feedback.affirmation_id -> scout.affirmations`). These are preserved per pre-flight Part 1's "no action" classification.
- **FKs on tables being dropped in migration 055** (e.g. `scout.task_templates.standard_of_done_id`, `scout.task_occurrences.task_template_id`, `scout.meal_transformations.family_id`). These drop with their source tables.
- **FKs into `public.meals`** held by `scout.meal_transformations` (`base_staple_id`, `transformed_staple_id`). Drop with the table in 055.
